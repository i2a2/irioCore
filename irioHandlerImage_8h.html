Configure CameraLink frame grabber.

Configure CameraLink frame grabber with the camera parameters. This configuration includes:
Set whether the CaneraLink signals FVAL, LVAL, DVAL, and SPARE are active high or active low.
Set whether the CameraLink control signals will be driven by the FPGA or not.
Set the scanning mode between regular and line scan.
Set the signal mapping when using extended configuration mode.
Set the configuration mode. This configuration must be done before running the FPGA.

Errors may occur if any of the ports were not found or while writing to the ports.

Parameters
[in]p_DrvPvtPointer to the driver session structure
[in]fvalHigh0=FVAL active low 1=FVAL active high
[in]lvalHigh0=LVAL active low 1=LVAL active high
[in]dvalHigh0=DVAL active low 1=DVAL active high
[in]spareHigh0=SPARE active low 1=SPARE active high
[in]controlEnable0=High impedance on conrol signals 1=Control signals driver by the FPGA.
[in]linescan0=Regular scan (complete image) 1=Line scan
[in]signalMappingSignal map to be used.
[in]configurationConfiguration to be used.
[out]statusWarning and error messages produced during the execution of this call will be added here.
Returns
TIRIOStatusCode result of the execution of this call.

Reads a message from the CameraLink camera.

Reads a message streamed from the CameraLink camera. This method blocks until a message is read.

The buffer data is an user allocated buffer of length data_size.

Errors may occur if any of the necessary ports was not found or while reading/writing from/to the ports.

Parameters
[in]p_DrvPvtPointer to the driver session structure
[in]data_sizeSize of previously allocated buffer (data) where message read will be stored
[out]dataPreviously allocated buffer where message read will be stored. Always ends in \0
[out]msg_sizeSize of the message read. This may be bigger than data_size. However, the data written to data will always be up to data_size bytes
